Activity

Activity 生命周期与启动模式

Activity生命周期

Activity的生命周期中方法说明：

  • onCreat():activity正在被创建，加载布局资源，加载数据
  • onStart():activity已经被创建，activity处于后台，暂不能和用户交互
  • onResume():activity可见，开始交互
  • onPause():onResume的反方法，activity不可见，到后台，可做一些不耗时的工作，如储存数据，结束动画
  • onStop():onStart的反方法，activity即将停止，可做一些不太重的回收工作
  • onDestroy():onCreat的反方法，activity将被销毁，进行数据的回收
  • onRestart():onStop后重新onStart时调用的方法

ps: 当一个activity开始另一个activity时，前一个先onPasue后，后一个的onCreate()才会调用;

Android Unit Test

单元测试对于一个应用来说是必要的，当测试一部分代码的时候，如果将整个项目编译运行，无疑是会浪费大量时间的。而采用单元测试对一部分代码进行测试，可以有效的节省时间。对于Android应用来说，由于其工程的特性，大部分逻辑都要通过界面来展示，所以大部分的测试都是直接跑真机或虚拟机。然而对于一些网络库，图片加载库等没有UI的项目，测试还要再写界面一来不必要，二来测试效率不高。由于Android使用的语言是Java，所以java中的单元测试可以使用到Android中来，然而Android中有着自己的一套构架，java单测不能完全使用，比如android中的一些特有类。。。解决办法由两种，一种使用模拟数据(Mock)，另一种是伪造一套AndroidSdk来进行测试(Robolectric 库)。这里主要谈及Mock

红黑树

红黑树

红黑树是AVL树的一个变种，最坏的情形花费时间为O(logN)。

红黑树性质

  1. 每一个节点或者染为红色或者染为黑色
  2. 根节点是黑色的
  3. 如果一个节点是红色的，那么他的子节点一定是黑色的
  4. 每个节点到一个NUll指针的每一条路径必需包含相同多数目的黑色节点

普通树的插入是直接将插入值作为树叶插入，但对于红黑树来说，这样的实现是不符合红黑树要求的。插入为黑色，则会违反条件4，插入为红色，对于父节点为红色的情况，会违反条件3

因而需要对颜色进行改变和对树进行旋转，使得插入后符合红黑树的性质
