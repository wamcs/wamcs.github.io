静态工厂方法代替构造器

静态工厂方法(static factory method)是一个返回实例的静态方法.eg:

advantage

  • 1. 静态工厂方法可以有不同的名称，代码更清楚，更易读。

    当我们进行构造器重载时，有时会遇到这样的情况。即我们通过赋给不同的形参来实现构造器的重载，但有可能两种计划不同的构造器由于使用了相同形似的形参使得编译产生错误。原因在于java对于每个方法都设定了方法签名(signature)这个东西，相当于ID，JVM通过方法签名来实现对不同方法的识别。方法签名构造规则是方法名加形参，方法返回类型并不作为签名考虑的范围，也可以理解方法重载时的限制了。因此，采用静态工厂方法一来可以解决这个问题，二来也能够更好的通过方法名来表明含义.

  • 2.静态工厂方法不必在每次调用它们的时候都创建一个新对象

    比如你可以在静态工厂方法里限定创建该对象的个数，当超出规定的个数时，返回缓存里的对象。这一点很显然，我们知道Singleton其实也提供一个静态工厂方法获取实例。除此之外，可以用==代替equals()方法，达到性能的提升。

  • 3.静态工厂方法可以返回原返回类型的任何子类型的对象

    构造器只能返回当前类的实例，无法返回子类的实例。

  • 4.静态工厂方法在创建参数化类型实例的时候使代码变得更加简洁。

    Java的泛型类在实例化时，还是需要写两次类型参数，非常冗长,利用静态工厂方法可以简化。

disadvantage

  • 1.没有提供public或者protect构造方法无法进行实例化
  • 2.和普通的静态方法相同，在API文档中没有像构造器一样明确的标识出来
