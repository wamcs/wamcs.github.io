Android Unit Test

单元测试对于一个应用来说是必要的，当测试一部分代码的时候，如果将整个项目编译运行，无疑是会浪费大量时间的。而采用单元测试对一部分代码进行测试，可以有效的节省时间。对于Android应用来说，由于其工程的特性，大部分逻辑都要通过界面来展示，所以大部分的测试都是直接跑真机或虚拟机。然而对于一些网络库，图片加载库等没有UI的项目，测试还要再写界面一来不必要，二来测试效率不高。由于Android使用的语言是Java，所以java中的单元测试可以使用到Android中来，然而Android中有着自己的一套构架，java单测不能完全使用，比如android中的一些特有类。。。解决办法由两种，一种使用模拟数据(Mock)，另一种是伪造一套AndroidSdk来进行测试(Robolectric 库)。这里主要谈及Mock

具体配置会在最下方给出,使用工具为Android Studio

JUnit 4

JUnit 4 是java单元测试工具。主要使用的是其注解

主要有这么几种：

  • @Before
  • @Test
  • @After
  • @Ingore
  • @BeforeClass
  • @AfterClass

其中Test标注的方法是真正测试的方法，别的除Ingore都是进行测试前的准备和测试后的回收。

@Before

  • junit测试类中的 任意一个测试方法 执行前 都会执行此方法

为测试方法初始化所需的资源。

@Test

Test有两种可选参数：

普通用法

异常测试：

参数expected代表测试方法期望抛出指定的异常，如果运行测试并没有抛出这个异常，则 JUnit 会认为这个测试没有通过

时间测试：

参数timeout指定被测试方法被允许运行的最长时间（ms），超时则 JUnit 认为测试失败。

@After

  • junit测试类中的 任意一个测试方法 执行后 都会执行此方法, 即使被 @Test 或 @Before 修饰的测试方法抛出异常

关闭由@Before注解修饰的测试方法打开的资源。

@Ingore

测试时忽略被该注释修饰的方法，当被测试的类的功能没写完时，可以跳过测试

@BeforeClass

  • 被修饰方法必需为 static 的
  • 只运行一次
  • Junit测试时第一个被运行的方法

这样的方法被用作执行计算代价很大的任务，如打开数据库连接。

@AfterClass

  • 被修饰方法必需为 static 的
  • 只运行一次
  • Junit测试时最后被运行的方法

给 @BeforeClass 扫尾

Mockito

Mockito 是一套Mock框架，是在frok EasyMock后进行修改完善的框架，也是google官方在Android Developer 里推荐unit test时使用的mock框架

Unit testing is defined as testing classes or methods in isolation. Java classes usually depend on other classes. A mock object is a dummy interface or class in which you define the dummy output of a certain method call. These objects can be provided to the class which should be tested to avoid any dependency to external data. The classical example is a mock object for a data provider. In production a real database is used but for testing a mock object simulates the database and ensures that the test conditions are always the same.

这是官方给出的Mock定义，Mock object既是一个伪造的class 或interface，用来为测试提供数据，保持测试的独立性。

既然拿不到数据，我们就伪造一份咯。

模拟方法调用

这个过程叫stub(官方命名)，主要用的方法是when(),then系列方法

模拟在调用某些方法时的行为，当执行某方法时，产生某些回应，return或是trow excption;thenAnswer()方法提供回调函数来设置比较复杂的逻辑。

参数的话可以使用matcher来设置这一类方法的行为，比如anyInt(),anyString()等一票方法。这里有一个限制就是，如果在调用方法时需要传入多个参数，其中一个参数使用了argument matcher，那么所有的参数必须都是matcher。不可以matcher和实际的参数混着用。如果是自己写的Matcher，那么需要调用argThat方法

验证方法

verify()方法用来验证方法到底有没有执行，执行了多少次。

spy

官方给出的样例

mock与spy的区别在于mock会改写写类的所有方法(如果是没有返回值的函数，则什么都不作，如果是有返回值，会返回默认值，比如布尔型的话返回false，List的话会返回一个空的列表，int的话会返回0等等)，而spy会保留类的方法，如果在某个方法中确认了返回一个值，那么spy是不会将其变为默认值的。同时mock支持的stub，spy同样也支持，而且效果相同。

用于模拟带有static方法的类，是对于Mockito的补充

使用

在任何需要用到 PowerMock 的类开始之前，首先我们要做如下声明：

@RunWith(PowerMockRunner.class)

然后，还需要用注释的形式将需要测试的类提供给 PowerMock：

@PrepareForTest( { YourClass.class })

静态方法模拟

@PrepareForTest( { YourClass.class })中YourClass为包含静态方法的类
