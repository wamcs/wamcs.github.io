Activity 生命周期与启动模式

Activity生命周期

Activity的生命周期中方法说明：

  • onCreat():activity正在被创建，加载布局资源，加载数据
  • onStart():activity已经被创建，activity处于后台，暂不能和用户交互
  • onResume():activity可见，开始交互
  • onPause():onResume的反方法，activity不可见，到后台，可做一些不耗时的工作，如储存数据，结束动画
  • onStop():onStart的反方法，activity即将停止，可做一些不太重的回收工作
  • onDestroy():onCreat的反方法，activity将被销毁，进行数据的回收
  • onRestart():onStop后重新onStart时调用的方法

ps: 当一个activity开始另一个activity时，前一个先onPasue后，后一个的onCreate()才会调用;

异常情况下Activity生命周期

1.系统相关资源配置发生改变，activity被杀死

activity会调用onSaveInstanceState(),保存数据调用时间是在onStop()之前。新的activity创建后系统会调用onRestoreInstanceState()方法，实现还原。调用时机在onStart之后。

每个View中也含有这两个方法，这是必要的。具体的保存和恢复流程如下：

Activity被意外杀死时，Activity会调用onSaveInstanceState保存数据，然后委托Window进行视图的保存。Window会委托DocerView来保存数据，DecorView再逐级委托。恢复也是相似的。在android中类似的思想还有，如View的绘制，触摸事件的分发等

2.系统资源不足

此时会按优先级来杀死activity，优先级从高到低依次为：

  • 前台显示的activity
  • 可见但非前台activity，如弹出dialog的情况
  • 后台activity

我们可以对activity的configChanges属性进行指定，已实现在某项内容发生改变时不需要重新创建activity

项目 含义
locale 设备的本地位置发生改变，一般是切换了系统语言
orientation 屏幕方向发生改变
keyboardHidden 键盘可访问性发生改变，例如用户调出键盘

上面三个是最为常用的

对于多个值可以使用“|”分割

activity启动模式

  • standard：标准模式，默认模式
  • singleTop：栈顶复用模式，当activity处于栈顶时，不会新创建activity，调用onNewIntent，否则仍会创建
  • singleTask：栈内复用，一个activity栈中只允许存在一个
  • singleInstance：单实例模式，系统会为其单独创建一个任务栈，除非这个任务栈被清除，不然不会创建新的activity

对于任务栈的设置，可以在androidManifest.xml中设置，activity下有一个参数TaskAffinity用于设置，默认是应用的包名。一般和singleTask启动模式或allowTaskReparenting属性配合使用。

若一个应用A启动了应用B中属性allowTaskReparenting为true的activity，那么当应用B启动时，该activity会从A的任务栈移到应用B的任务栈中

启动模式的设置

有两种方式，一种是在AndroidManifest.xml中设置，属性launchMode，另一种是给Intent设flag，第二种的优先级高于第一种。

给出几种Flag：

  • FLAG_ACTIVITY_NEW_TASK
  • FLAG_ACTIVITY_SINGLE_TOP
  • FLAG_ACTIVITY_CLEAR_TOP
  • FLAG_ACTIVITY_CLEAR_TASK

FLAG_ACTIVITY_NEW_TASK有坑，并不是和官方说的和launchMode = “singleTask”一样，不会把其上面的activity出栈，需要和FLAG_ACTIVITY_CLEAR_TOP配合食用

IntentFilter匹配规则

应对于隐式启动，即不给明需要启动的activity，，广播中用的比较多，activity最明显的栗子就是应用开启时最先展现的activity，想想AndroidManifest.xml对于MainActivity所加的IntentFilter。

  • action
  • category
  • data

IntentFilter可以有多组，满足其中一组就可以启动

action

action可以自己设置，也可以使用系统的，Intent中的action必需与过滤规则中的action吻合才能启动，区分大小写。至少要有一个

category

只有当Intent请求中所有的Category与组件中某一个IntentFilter的完全匹配时，才会让该 Intent请求通过测试，IntentFilter中多余的声明并不会导致匹配失败

data

由两部分组成，URI和mimeType，mimeType指媒体类型，例如image/jpeg、audio/mpeg4-generic和video/* 表示图片，文本，视频等不同的媒体格式。

Intent的设置方法中setData和setType都会分别设置Data和Type，一起使用前一个设置的内容会被抹掉，要全部加的话使用setDataAndType方法
