历遍

三种历遍方式的不同主要在于根节点的历遍时间

  • 先序历遍：根节点 -> 左子树 -> 右子树
  • 中序历遍：左子树 -> 根节点 -> 右子树
  • 后序历遍：左子树 -> 右子树 -> 根节点

虽然所有的递归都可以改为非递归，但毕竟递归简洁啊，如此就必定要牺牲别的比如时间

普通树

这样父节点的FirstChild指向其第一个子节点，而另一个NextSibling指向与其相邻的第一个兄弟节点，这样将整个树串起来

一个与（一）中对应的应用是是表达式树

此树的全部父节点都是运算符号，子节点为数字，用中序历遍打印出运算式，构造也很简单，和栈的使用一样，只不过遇到符号是使之变为父节点，其子节点为栈中pop出的俩数，然后把构造了的树再入栈

二叉树

由于其查找为二分查找，故运行时间平均为O ( log N )

AVL树

Avl树保持树的深度，即左子树与右子树的最大深度相差不能超过一，与二叉树的区别主要在于在insert与delete中加入旋转操作，使得树保持平衡
